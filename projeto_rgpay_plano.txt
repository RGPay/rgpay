# PLANO DE DESENVOLVIMENTO RGPAY - PROTÓTIPO RÁPIDO

## VISÃO GERAL DO PROJETO
Sistema de modernização para estabelecimentos comerciais que utilizam maquinetas Cielo Lio, oferecendo gerenciamento de cardápio, registro de pedidos, painel administrativo, e suporte multi-unidades.

## RESTRIÇÕES E PARÂMETROS
- Equipe de apenas 2 desenvolvedores
- Prazo de 1-2 semanas para protótipo funcional
- Versão mobile como PWA (Progressive Web App)
- Foco em funcionalidades essenciais primeiro

## ESTRATÉGIA DE DESENVOLVIMENTO RÁPIDO COM IA

### 1. ESTRUTURAÇÃO INICIAL (Dias 1-2)
1.1. Scaffolding do projeto React + TypeScript + NestJS
1.2. Estrutura de pastas e componentização básica
1.3. Configuração mínima de CI/CD
1.4. Modelos essenciais de dados (foco em Produto, Pedido, Usuário)

### 2. AUTENTICAÇÃO E CORE UI (Dias 3-4)
2.1. Sistema simples de login/autenticação
2.2. Layout principal responsivo (compatível com PWA)
2.3. Componentes de UI reutilizáveis (tabelas, cards, inputs)
2.4. Navegação básica entre telas

### 3. FUNCIONALIDADES PRIORITÁRIAS (Dias 5-7)
3.1. Cadastro de produtos/cardápio (CRUD básico)
3.2. Interface de listagem de pedidos
3.3. Gestão simplificada de unidades
3.4. Dashboard com métricas essenciais

### 4. INTEGRAÇÃO MÍNIMA COM MAQUINETAS (Dias 8-10)
4.1. Mock da API Cielo para protótipo
4.2. Fluxo de pedido simplificado
4.3. Geração de comprovante básico

### 5. FINALIZAÇÃO E DEPLOY (Dias 11-14)
5.1. Configuração de PWA
5.2. Deploy em ambiente Azure
5.3. Testes de aceitação básicos
5.4. Documentação essencial

## ABORDAGEM DE DESENVOLVIMENTO PARA EQUIPE PEQUENA

### DIVISÃO DE TRABALHO RECOMENDADA
1. **Desenvolvedor 1**: Foco em frontend, componentização e UI/UX
2. **Desenvolvedor 2**: Foco em backend, APIs e integração

### ESTRATÉGIAS PARA MAXIMIZAR PRODUTIVIDADE
1. **Usar IA para accelerar desenvolvimento**:
   1.1. Geração de componentes de UI repetitivos
   1.2. Implementação de modelos e controllers padronizados
   1.3. Configuração de ambiente e scaffolding

2. **Priorizar MVPs por funcionalidade**:
   2.1. Implementar versão mínima de cada feature
   2.2. Postergar refinamentos para versões futuras
   2.3. Foco em fluxos críticos do negócio

3. **Comunicação e Integração**:
   3.1. Reuniões diárias rápidas (15min)
   3.2. Compartilhamento de componentes via padrões comuns
   3.3. Documentação just-enough para interfaces críticas

## ESCOPO MÍNIMO VIÁVEL PARA PROTÓTIPO

### 1. FRONTEND ESSENCIAL
1.1. Tela de login/autenticação simplificada
1.2. Dashboard com KPIs básicos
1.3. Cadastro e listagem de produtos
1.4. Visualização de pedidos
1.5. PWA básico para acesso mobile

### 2. BACKEND ESSENCIAL

2.2. CRUD de produtos e pedidos
2.3. Mock de integração com maquinetas
2.4. API para as telas principais

### 3. INFRAESTRUTURA MÍNIMA
